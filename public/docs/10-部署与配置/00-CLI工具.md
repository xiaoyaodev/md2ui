# CLI 工具

验证 md2ui 命令行工具的各项功能。

## 基本用法

```bash
# 在文档目录下启动预览
cd /path/to/your/docs
md2ui

# 指定端口
md2ui -p 8080

# 静态构建
md2ui build
```

## 启动流程

1. 扫描当前目录下的 `.md` 文件
2. 加载用户配置文件（如果存在）
3. 启动 Vite 开发服务器
4. 自动打开浏览器

## 配置文件加载

CLI 按以下顺序查找配置文件：

| 优先级 | 文件名 | 格式 |
|--------|--------|------|
| 1 | `md2ui.config.js` | ES Module |
| 2 | `.md2uirc.json` | JSON |

命令行参数优先级最高，会覆盖配置文件中的同名配置。

## 服务端 API

开发服务器提供以下 API：

| 路径 | 方法 | 说明 |
|------|------|------|
| `/@user-docs-list` | GET | 文档列表 |
| `/@site-config` | GET | 站点配置 |
| `/@user-docs/{path}` | GET | 文档内容 |

## 验证要点

1. 在包含 .md 文件的目录下运行 `md2ui` 应启动服务
2. 无 .md 文件时应提示错误并退出
3. `-p` 参数应正确设置端口
4. 配置文件应被正确加载
