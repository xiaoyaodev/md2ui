# 编辑器功能方案

## 一、需求概述

在现有文档查看页基础上，新增编辑页，实现两个页面并行：

- **查看页**（现有）：负责美观展示、SEO、网页分发
- **编辑页**（新增）：负责良好的 Markdown 编辑体验

核心要求：
1. 所见即所得编辑，支持 Markdown 语法（非左右分栏）
2. Mermaid 图表：默认显示图片，点击可编辑代码
3. 两个页面 UI 风格一致，页面上按钮可切换
4. 左侧菜单和当前文档内容支持实时刷新

## 二、技术决策

| 决策项 | 选型 | 理由 |
|--------|------|------|
| 编辑器 | Tiptap | Vue 3 一等支持，ProseMirror 底层，定制性强，样式可控，与"两页 UI 一致"需求匹配 |
| 后端 | Vite 插件 | 需求简单（保存文件 + 文件列表），不需要独立框架，零额外依赖 |
| 前端框架 | 保持 Vue 3 | 现有 12 个组件 + 10 个 composables，重写成本远大于收益 |
| 实时刷新 | HTTP 轮询 + ETag | 实现最简，1 秒延迟够用，ETag 优化后无效请求开销极低 |

## 三、架构设计

### 3.1 路由方案

使用 hash 模式路由，兼容现有静态部署：

```
/#/view/{docHash}     → 查看页（现有逻辑）
/#/edit/{docHash}     → 编辑页（新增）
```

### 3.2 页面结构

两个页面共享：侧边栏、TOC、TopBar、移动端适配、文档树管理。


## 四、编辑器设计

### 4.1 Tiptap 集成

使用 Tiptap + tiptap-markdown 扩展，实现所见即所得编辑：

- 加载时：Markdown 源文本 → Tiptap 文档模型
- 保存时：Tiptap 文档模型 → 标准 Markdown 文本
- 编辑时：用户在富文本界面操作，也可输入 Markdown 语法自动转换

### 4.2 新增依赖

```json
{
  "@tiptap/vue-3": "^2.x",
  "@tiptap/starter-kit": "^2.x",
  "@tiptap/extension-table": "^2.x",
  "@tiptap/extension-table-row": "^2.x",
  "@tiptap/extension-table-cell": "^2.x",
  "@tiptap/extension-table-header": "^2.x",
  "@tiptap/extension-task-list": "^2.x",
  "@tiptap/extension-task-item": "^2.x",
  "@tiptap/extension-image": "^2.x",
  "@tiptap/extension-code-block-lowlight": "^2.x",
  "tiptap-markdown": "^0.8.x"
}
```

### 4.3 Mermaid NodeView

自定义 Tiptap NodeView 组件，两种状态切换：

- **预览态**（默认）：调用 `mermaid.render()` 显示 SVG 图表，图表上方显示编辑按钮
- **编辑态**（点击编辑）：显示代码编辑区域，支持语法高亮，失焦或点击确认后重新渲染图表

### 4.4 工具栏

浮动工具栏（选中文本时出现）+ 顶部固定工具栏：

- 基础格式：加粗、斜体、删除线、行内代码
- 标题：H1 ~ H4
- 列表：有序、无序、任务列表
- 插入：代码块、表格、图片、链接、Mermaid 图表
- 操作：撤销、重做、保存

### 4.5 Markdown 序列化

- 使用 `tiptap-markdown` 扩展处理双向转换
- 保留 frontmatter（title、description 等元数据）
- Mermaid 代码块序列化为标准 ` ```mermaid ... ``` ` 格式

## 五、实时刷新设计

### 5.1 方案：HTTP 轮询 + ETag

```
前端每 1s → GET /api/docs-list（带 If-None-Match）
           → 304：无变化，跳过
           → 200：文件列表变了，更新侧边栏菜单

前端每 1s → GET /api/doc-content?path=当前文档路径（带 If-None-Match）
           → 304：无变化，跳过
           → 200：内容变了，重新渲染当前文档
```

### 5.2 服务端 API（Vite 插件）

```
GET  /api/docs-list              → 返回文件列表 + ETag
GET  /api/doc-content?path=xxx   → 返回文件内容 + ETag
POST /api/save                   → 保存文件内容
```

- ETag 基于文件列表/内容的 hash 生成
- 没变化时返回 304，零传输开销
- 只在开发模式（Vite dev server）下生效

### 5.3 前端轮询逻辑（useFileWatcher.js）

- 组件挂载时启动轮询，卸载时停止
- 文件列表变化 → 调用 `reloadDocsList()` 刷新侧边栏
- 当前文档变化 → 查看页调用 `renderMarkdown()` 重新渲染
- 当前文档变化 → 编辑页弹出提示"文件已被外部修改，是否加载最新内容？"
- 请求失败时静默忽略，下一秒自动重试

### 5.4 编辑冲突处理

当用户在编辑页编辑文档，同时外部（AI）也修改了同一文件：

- 如果编辑器内容未被用户修改（干净状态）→ 静默更新
- 如果编辑器内容已被用户修改（脏状态）→ 弹出提示，用户选择"加载最新"或"忽略"

## 六、样式一致性

- 编辑页的 Tiptap 编辑区域复用查看页的 `.markdown-content` CSS 样式
- 共享 CSS 变量（颜色、字体、间距）
- 编辑器额外样式（光标、选区、工具栏）单独定义
- 代码块、表格、Mermaid 图表的渲染样式两页保持一致
